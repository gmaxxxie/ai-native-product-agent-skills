# 中段判断 Skill

## 适用场景
- 确定产品形态与交付形式
- 评估用户信任与采用意愿
- 设计人机协作方案

## 输入
| 字段 | 说明 |
|------|------|
| target_users | 目标用户画像 |
| solution_concept | 解决方案概念 |
| ai_role_options | AI角色选项（助手/协作者/代理等） |

## 输出
- 产品形态建议（产品/模块/能力层）
- 用户托付度评估
- 人机协作设计方案

## 工作流程
1. **形态判断**：做成独立产品、系统模块，还是工作流中的能力层？
2. **托付度判断**：用户愿意尝试吗？愿意持续用吗？敢把关键任务交给它吗？
3. **协作方式判断**：AI该辅助还是代理？模型边界在哪里？需要补哪些交付条件？

## 注意事项
- 形态选择影响技术架构与商业化路径
- 托付度是产品能否成立的关键——再好的技术，用户不敢用等于零
