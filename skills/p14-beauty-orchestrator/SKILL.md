# AI Beauty 编排器（Beauty Orchestrator）

## 适用场景
- 设计师/产品经理想建立系统性的审美能力
- 需要理解AI时代的美感本质与价值
- 想学习如何与AI协作保持个人审美主权

## 输入
| 字段 | 说明 |
|------|------|
| user_goal | 用户的审美相关目标（如"提升美感""定义标准"） |
| current_level | 当前审美水平（入门/中级/高级） |
| ai_usage_context | 使用AI的场景（生成/评估/协作） |

## 输出
- 推荐Skill调用链（按美感五阶段）
- 各阶段重点训练清单
- 个性化审美提升路线图

## 工作流程
1. **目标分类**：判断用户目标属于美感体系的哪个阶段
2. **构建学习链**：按"认知→能力→人机→标准→护城河"顺序串联
3. **动态裁剪**：根据用户当前水平，跳过或深化某些环节
4. **输出路线图**：生成结构化的审美能力提升计划

## 美感五阶段
1. **认知重塑**（Ch1-3）：理解生成时代美感的本质
2. **能力拆解**（Ch2）：掌握六维美感模型
3. **AI协作**（Ch4-7）：学会与AI协同创作
4. **标准建立**（Ch16-18）：形成不可复制的个人标准
5. **护城河巩固**（Ch8-11）：在AI时代保持人的优势

## 注意事项
- 美感不是"感觉"，是可训练的系统能力
- 最终目标是建立"审美主导权"，而非依赖AI生成

## 核心概念

### 概念一：生成焦虑与判断稀缺

当AI让"做出来"变得容易，过剩就成了新的常态。团队面对几十个版本，常常不是没东西可选，而是不知道该选哪个。真正稀缺的不再只是产出能力，而是一种更难被替代的能力——美感。美感不是消失了，而是从结果层上升到了主导层：在作品出现之前就已经介入，定义方向、排除不值得的选项、在过多的可能性里建立秩序。

### 概念二：美感六维度模型

美感不是一种整体的、模糊的直觉，而是一套可以被分层理解、逐层练习的能力系统：
- **感受力**：能不能感受到差异——美感的入口
- **判断力**：能不能说清楚差异——把直觉变成语言
- **选择力**：能不能在大量可能中选出最准的——AI时代最关键的一环
- **平衡力**：能不能在限制中仍然成立——美感的实战维度
- **故事力**：能不能让形式背后有生命——美感的深度维度
- **一致性**：能不能在不同场景中保持水准——美感的稳定性

### 概念三：美感双轴模型（艺术性 × 商业性）

美感有两个独立维度：向内的艺术性（个人表达、辨识度）和向外的商业性（用户适配、市场接受度）。真正值钱的位置在第四象限——高表达 × 高适配。AI擅长帮你向右移（接近用户），但向上走（提升个人表达立场）需要你自己完成。

### 概念四：AI是放大器、陪练器、协作者

AI不会自动提升你的审美，它只会放大你已有的审美。标准模糊的人用AI会得到大量"还行"但不成立的输出；标准清晰的人用AI则会获得极高的迭代效率。AI的四个角色：放大器（扩展能力边界）、陪练器（加速感受力训练）、转译器（缩短感受到做出的距离）、协作者（参与思考但不替代最终判断）。

### 概念五：审美主导权与持续进化

真正的护城河不是保密，而是进化速度。当别人学会你的风格，你的标准已经往前走了。最终难以替代的不是某个作品、某种风格或某个技巧，而是那个能不断校准、不断提高标准的人。审美主导权 = 对美感标准有定义权 + 能主导什么值得被创造 + 不会被工具或趋势牵着走。

## 分步执行

### 步骤一：目标分类与阶段定位

识别用户当前的审美相关目标属于美感体系的哪个阶段：
1. 收集用户的 `user_goal`（审美相关目标）
2. 评估 `current_level`（入门/中级/高级）
3. 了解 `ai_usage_context`（生成/评估/协作场景）
4. 将目标映射到五个阶段：认知重塑 → 能力拆解 → AI协作 → 标准建立 → 护城河巩固

**输出**：阶段定位报告，包含当前水平评估和目标差距分析。

### 步骤二：构建学习链

按"认知→能力→人机→标准→护城河"顺序串联Skill调用链：
1. **认知重塑阶段**：调用 p14h（前言命题）→ p14a（美感重定义）
2. **能力拆解阶段**：调用 p14a（六维度模型）→ p14d（故事力训练）
3. **AI协作阶段**：调用 p14b（AI角色定位）→ p14c（选择力与Context）
4. **标准建立阶段**：调用 p14f（商业化验证）→ p14g（系统训练）
5. **护城河巩固阶段**：调用 p14e（人的优势与进化）

**输出**：结构化的Skill调用链和推荐顺序。

### 步骤三：动态裁剪

根据用户当前水平，跳过或深化某些环节：
1. 如果用户已经是高级水平，跳过认知重塑，直接进入标准建立
2. 如果用户主要困惑在AI协作，聚焦 p14b + p14c
3. 如果用户面临商业化问题，聚焦 p14f + p14g
4. 如果用户需要快速建立信心，从 p14e（人的优势）开始

**输出**：个性化的精简学习路径。

### 步骤四：输出路线图

生成结构化的审美能力提升计划：
1. 按周/月维度拆解训练任务
2. 每个阶段明确：要读的章节、要做的练习、要达到的标准
3. 设置检查点：每个阶段结束时的标准校准
4. 提供AI协作的具体工作流模板

**输出**：个性化审美能力提升路线图，含时间表和检查标准。

### 步骤五：执行跟踪与标准校准

在执行过程中持续校准：
1. 每完成一个阶段，回顾输出质量是否提升
2. 用"如果现在重新做，我会怎么做"来检验标准是否在提高
3. 记录判断语言的变化——从模糊感觉到清晰表述的进展
4. 根据实际反馈调整后续路线

**输出**：阶段复盘报告和路线调整建议。

## 示例 1

**场景**：一个产品经理刚接触AI生成工具，感到审美焦虑，想系统提升美感。

**编排器执行过程**：

**输入**：
```
user_goal: "用AI做产品设计时，总觉得输出不够好但说不清为什么"
current_level: 入门
ai_usage_context: 生成+评估
```

**步骤1 - 阶段定位**：
用户处于"认知重塑"阶段，核心问题是审美标准模糊，感受力已具备但判断力不足。需要从基础认知开始。

**步骤2 - 学习链构建**：
```
p14h（理解核心命题：判断稀缺）
  → p14a（建立六维度认知框架）
    → p14b（明确AI是放大器，不是替代者）
      → p14c（学习Context构建和选择力）
        → p14g（建立系统训练习惯）
```

**步骤3 - 动态裁剪**：
- 跳过 p14f（商业价值），因为用户当前问题不在商业化
- 深化 p14a 的"感受力→判断力"过渡
- 增加 p14c 的Context构建实操

**步骤4 - 输出路线图**：
```
第1周：阅读前言+Ch1，理解"生成焦虑"的本质
       练习：收集5个你觉得"不够对"的AI输出，尝试说清"哪里不对"
第2周：阅读Ch2，建立六维度框架
       练习：用六维度分析3个你喜欢的产品
第3周：阅读Ch4，理解AI的正确角色
       练习：为同一个需求分别用"命令式"和"脉络式"给AI指令，比较结果
第4周：阅读Ch5-6，学习Context构建
       练习：为你的产品写一份完整的Context文档
```

**输出**：四周审美提升路线图，含每日练习和周检查标准。

## 示例 2

**场景**：一个品牌设计师已经有不错的审美基础，想建立不可复制的个人标准体系。

**编排器执行过程**：

**输入**：
```
user_goal: "想建立自己的审美标准体系，让作品有持续的辨识度"
current_level: 高级
ai_usage_context: 协作+评估
```

**步骤1 - 阶段定位**：
用户处于"标准建立→护城河巩固"过渡阶段，感受力和判断力已成熟，需要的是标准显性化和持续进化机制。

**步骤2 - 学习链构建**：
```
p14c（强化选择力，从"选好看"到"选成立"）
  → p14e（理解人的不可替代性）
    → p14g（建立系统训练和标准进化机制）
      → p14f（用商业价值验证标准）
```

**步骤3 - 动态裁剪**：
- 跳过 p14h、p14a、p14b（基础认知已具备）
- 深化 p14e 的"标准不可复制"理论
- 重点训练 p14g 的"标准显性化"和"AI协作流程"
- 增加 p14f 的"美感溢价"案例分析

**步骤4 - 输出路线图**：
```
第1周：阅读Ch10，理解"作品可复制，标准不能"
       练习：回顾你过去一年的作品，列出你反复坚持的5个审美判断
第2周：阅读Ch17，学习"让输出更像你"的协作方法
       练习：建立你的风格库（参照物/偏好/禁忌/气质语言）
第3周：阅读Ch16，设计你的系统训练计划
       练习：用AI做对比练习、选择练习、极限练习各一次
第4周：阅读Ch18，理解"持续提高标准"的方法
       练习：为你3个月前的作品写复盘，记录标准的变化
```

**输出**：四周标准体系构建路线图，含风格库模板和标准校准机制。

## Skill 调用决策树

根据用户输入的关键词，快速路由到对应 Skill：

```
用户提到                          → 推荐 Skill
────────────────────────────────────────────────
"不知道怎么选"/"选择困难"          → p14c（选择力与Context）
"AI做的东西没有辨识度"/"都差不多"  → p14b（AI角色定位）+ p14d（故事力）
"怎么写好prompt"/"输出不稳定"     → p14c（Context构建）
"怕被AI替代"/"我的价值在哪里"     → p14e（人的优势）
"美感怎么训练"/"怎么提升品味"     → p14g（系统训练）
"怎么说服老板美感值钱"/"ROI"      → p14f（商业价值）
"想建立个人风格"/"让输出更像我"   → p14b（AI协作）+ p14g（风格库）
"什么是美感"/"美感到底是什么"     → p14a（美感重定义）
"快速了解这本书"/"给我一个概览"   → p14h（前言命题）
```

## 常见场景速查

### 场景 A：新手入门（从未系统思考过美感）
```
路径：p14h → p14a → p14b → p14c → p14g
总时长：约 4-6 周
每周投入：3-5 小时
关键产出：个人审美标准文档（初稿）+ 选择力Checklist
```

### 场景 B：进阶提升（有基础但想突破瓶颈）
```
路径：p14c → p14d → p14e → p14g
总时长：约 3-4 周
每周投入：2-3 小时
关键产出：风格库 + 故事力训练方案 + 标准进化机制
```

### 场景 C：商业化落地（想把美感转化为商业价值）
```
路径：p14f → p14g → p14e
总时长：约 2-3 周
每周投入：3-4 小时
关键产出：美感ROI分析 + 体验美感评估 + 护城河方案
```

### 场景 D：团队推广（想让团队建立审美共识）
```
路径：p14h（全员）→ p14a（全员）→ 分组学习 → p14g（全员）→ 实战
总时长：约 6-8 周
每周投入：团队 1-2 小时 + 个人 2-3 小时
关键产出：团队审美共识文档 + 个人标准文档 + 团队协作规范
```
