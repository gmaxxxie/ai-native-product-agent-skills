# 九步闭环总论 Skill

## 适用场景
- 面对AI产品决策缺乏完整框架
- 需要覆盖产品决策全生命周期
- 协调跨职能团队共同决策

## 输入
| 字段 | 说明 |
|------|------|
| product_idea | 产品/功能想法 |
| decision_stage | 当前处于哪个决策阶段（如不清楚则输入"初始"） |

## 输出
- 九步闭环路线图
- 当前阶段的核心任务
- 上下游衔接点

## 工作流程
1. **九步认知**：向用户解释九步闭环的完整框架
2. **阶段定位**：帮助用户识别当前所处阶段
3. **衔接说明**：明确与前后阶段的输入/输出关系
4. **下一步指引**：指向对应的具体Skill

## 九步框架
1. 做什么：先看清自己到底在拍什么板
2. 值不值得做：这个问题值不值得投入资源优先解决
3. 用不用AI：这个问题该不该用AI做
4. 做成什么：是产品、模块，还是嵌进工作流的一层能力
5. 用户会不会托付：用户愿不愿意试、愿不愿意持续用、敢不敢把关键任务交给它
6. 怎么做：任务该辅助还是托付，模型边界怎么看，系统要补哪些交付条件
7. 怎么验证：看哪些行为、信号和指标，才能判断不是demo成立，而是真成立
8. 怎么进入市场：先卖给谁、怎么定价、怎么发布、怎么让销售营销接住
9. 反馈回路：市场反馈后如何进入反馈回路，持续校准判断

## 注意事项
- 九步是理想状态，现实决策可能跳跃或并行
- 每一步的产出都是可交付的"判断记录"
